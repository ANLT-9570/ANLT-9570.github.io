多线程的基本使用

线程的状态

传统的操作系统线程模型中，线程被分为五中状态，在java线程中，线程被分为六种，各分别为：

操作系统：新建(NEW)，就绪(RUNNABLE)，运行(RUNNING)，阻塞(BLOCK)，死亡(DEAD)

java层面：新建(NEW)，可运行(RUNNABLE)，阻塞(BLOCK)，等待(WAITING)，超时等待(TIME_WATING)，终结(TERMINATED)

Thread常用api了解

线程池

线程池是一种用于管理和复用线程的机制，可以避免线程的频繁创建和销毁，提高利用率，优化系统的性能和资源消耗。

线程池的创建方式：共7种(6种是通过Executors创建的，1中是通过ThreadPoolExecutor创建的)；

线程池数量设置

任务的类型分为：IO密集型和CPU密集型

IO密集型：频繁读取磁盘上的数据，或者需要通过网络远程调用接口

CPU密集型：非常复杂的调用，循环次数很多，或者递归调用层次很深等

IO密集型配置线程数：2N，其中N代表CPU核数。

CPU密集型配置线程数：N + 1，其中N代表CPU核数。

可通过Runtime.getRuntime().availableProcessors()获取CPU核心数
